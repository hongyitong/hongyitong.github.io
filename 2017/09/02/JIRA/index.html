JIRA

发表于 2017-09-02 | 分类于 技术相关 | 浏览 次

我们的技术团队有25人。最近一年正好把管理系统换成JIRA+GIT+Confluence.
用户体验十分好，简单易用直观，省了很多对非技术人员的培训成本。技术人员用得舒心，管理层的效率也提高了不少。有两点比较Impressive。

  1. 代码+Issue+wiki无缝连接三个平台基本上都能实现无缝连接。特别当JIRA跟GIT结合起来后 。代码管理有了很大的提高。能从JIRA ISSUE里面快捷地看到代码的改动。工作量一目了然。对管理层来说，非常高效。
  2. Jira - Agile直观显示安装Jira Agile后。对Scrum项目是一大福音。里面Drap and drop的Kanban图，操作十分方便。还有burn down chart。哪里快了哪里慢了，项目进度清晰明了。
    之前比较了很多软件，这个组合是我们最后的选择。Jira+Git+Confluence 强力推荐！

我们公司一直都是confluence和jira一起用的，confluence作为wiki管理需求，jira管理任务、进度和bug。 这两个工具的组合对产品管理来说并不是最理想，但还可以用。Jira本身不是一个管理需求的工具，但是如果项目不太复杂还行。几点建议:

  1. 把ticket按照需求， bug, 变更，问题，风险等分类管理。
  2. 用jira的link功能和每个user story或任务的唯一编码，关联所有相关的ticket
  3. jira有一些插件，可以生成甘特图，和SVN整合使用，建立从需求/bug到代码的联系

Jira是一个issue管理工具，可以管理需求、Bug、任务等。可以做一个备选的工具，但以我的经验来看，更多的还是在其它的管理方面做好。不知道你的团队有多大？分享一点我的经验（10人团队的）

  1. 及时沟通：每天早上的站立会议，每周的项目例会，及时的需求及技术讨论会议，这些是必不可少的；特别是每天早的站立会议，让团队明白当关的状态在哪，今日的目标是什么，每个团队成员的承诺是什么，有什么依赖，有什么风险；站立会议很重要；
  2. 目标与计划：需要做好大的计划点，每一轮的迭代计划，关键的里程碑；更重要的是下一周的计划，我一般只会细化到下一周的计划；让团队成员清楚，我们的大目标在哪，当前的目标是什么；
  3. 展示与激励：及时的展示团队成员的成果，一可以增加开发者的信心与成就感，二可以提高开发者的质量；做好每周的项目红榜，对于做的好的成员，给予认可，发点小礼品并全体团队成员为之鼓掌；
  4. 简单的工具：工具一定要简单，不要太复杂，比如用excel跟踪需求，bug等，我们以前还用hdwiki做过需求、方案文档的写作。
  5. 关键利益人的沟通：除了团队成员外，与你团队相关的关键利益人要沟通好，比如业务方的领导，项目相关的人员等，要管理好这些人的期望；
  6. 张弛结合：每两周做为一个冲锋期，这段时间要让大家紧张起来，目标达成，结果性强点；两周后适当的放松几天，让团队成员感觉到工作轻松点。
