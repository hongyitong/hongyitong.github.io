虚拟机漂移导致网络延时的问题

发表于 2018-06-23 | 分类于 工作相关

【现象】有一台BPM服务器突然延时很厉害，在机房内ping的延时都是100ms以上，但是在同一个交换机端口上的其它服务器ping值小与1ms；
【分析】后来分析，看到这个BPM服务器发生过漂移，漂移到了另外一台物理服务器，而另外一台物理机的网卡设置问题，需要从Linux 虚拟机网络适配器从E1000改为VMXNET3
【原理】
我们知道VMware的网络适配器类型有多种，例如E1000、VMXNET、VMXNET 2 (Enhanced)、VMXNET3等，就性能而言，一般VMXNET3要优于E1000，下面介绍如果将Linux虚拟机的网络适配器类型从E1000改为VMXNET3。

tcpdump与Wireshark介绍（网络抓包工具）

发表于 2018-06-16 | 分类于 技术相关

本文只是转载的一部分内容，具体的内容见以下链接。
http://www.cnblogs.com/jhj117/p/7827331.html

1 起因

前段时间，一直在调线上的一个问题：线上应用接受POST请求，请求body中的参数获取不全，存在丢失的状况。这个问题是偶发性的，大概发生的几率为5%-10%左右，这个概率已经相当高了。在排查问题的过程中使用到了tcpdump和Wireshark进行抓包分析。感觉这两个工具搭配起来干活，非常完美。所有的网络传输在这两个工具搭配下，都无处遁形。
为了更好、更顺手地能够用好这两个工具，特整理本篇文章，希望也能给大家带来收获。为大家之后排查问题，添一利器。

2 tcpdump与Wireshark介绍

在网络问题的调试中，tcpdump应该说是一个必不可少的工具，和大部分linux下优秀工具一样，它的特点就是简单而强大。它是基于Unix系统的命令行式的数据包嗅探工具，可以抓取流动在网卡上的数据包。
默认情况下，tcpdump不会抓取本机内部通讯的报文。根据网络协议栈的规定，对于报文，即使是目的地是本机，也需要经过本机的网络协议层，所以本机通讯肯定是通过API进入了内核，并且完成了路由选择。【比如本机的TCP通信，也必须要socket通信的基本要素：src ip port dst ip port】
如果要使用tcpdump抓取其他主机MAC地址的数据包，必须开启网卡混杂模式，所谓混杂模式，用最简单的语言就是让网卡抓取任何经过它的数据包，不管这个数据包是不是发给它或者是它发出的。一般而言，Unix不会让普通用户设置混杂模式，因为这样可以看到别人的信息，比如telnet的用户名和密码，这样会引起一些安全上的问题，所以只有root用户可以开启混杂模式，开启混杂模式的命令是：ifconfig en0 promisc, en0是你要打开混杂模式的网卡。

1、信息化项目的招标方式

现状：有较多项目都是招标之后，另行报告，采用非价低者。
建议：参照业内同行做法，信息化招标方式可以考虑分技术标、商务标两个环节，主要的目的是提高公司在招标环节的议价能力。
方式：技术标针对投标单位的技术符合度、需求符合度进行打分，提前制定相关的技术参数、需求功能点以及合格的分数。技术和业务参与打分，合格的进入商务标环节。
商务标原则上采取价低者得，有些技术要求比较高的，可以提前设定技术和价格不同权重，进行综合评分，高分者得。

推广IPG的反思

发表于 2018-06-09 | 分类于 地产相关

主要是工作方式的问题，还是太急于求成，想出成绩导致，其实还是需要稳扎稳打。所以说，推广的时候，还是要根据推广的结果、对大家的影响、大家的反应来提前准备应对策略，类似风险管理的内容。
其实在技术上考虑最不周的问题是，在实施前，想到了用户使用习惯的反弹，准备好了KK、邮件、网盘来替代微信、QQ，但没有考虑到禁用微信、QQ之后，文件都在内网跑，把总部的带宽挤爆了。
不过最大的收获还是，通过这个项目，自己得到了很多反思。也明白了一个重要的道理，既然做了就要坚持下去，不要放弃。同时也明白了两个原则，那就是，以公司的管控制度为基本的判断标准、学会承受压力和过滤噪音。在工作层面，这个项目至少还是把大家的外部邮箱全部禁用了，使大家习惯了使用企业邮箱对外进行沟通。

1、和入域的项目在一起推

IPG、入域都是影响用户桌面，两者放一起导致，工作量剧增，另外，出问题之后，搞不清是IPG出的问题，还是入域出的问题。
例如，在NC启动BPM流程出现卡顿的问题时，不知道是因为IPG启动策略影响还是入域的策略导致影响，并且，两者启动之后带来的问题混合在一起，增加了很多事件量。
 补充说明，后来关于卡顿问题，最终发现是因为项目公司连接总部的入口带宽不够导致。主要是当时没有考虑到IGP策略禁用微信、QQ传输文件之后所有的文件传输通过KK、企业邮箱，所有这些流量都是走内部的带宽，导致总部连接项目公司的带宽的接口使用量暴增，从实施前的30M左右升到了50M左右，而整个带宽才50M，所以经常出现连接的问题，并且只有项目公司出现，并且是有时有问题，过一会又好了。
